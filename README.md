## 🎯 What Is This Tool?

**ECLYPSE Facilities Alarm Console** is a **power-user alarm investigation and operations utility** designed to improve visibility and control of alarms generated by ECLYPSE Facilities controllers.

It provides a centralized desktop interface for:
- Viewing alarms across multiple controllers
- Filtering, searching, and paging large alarm datasets
- Automatically polling alarm endpoints
- Acknowledging alarms individually or in bulk
- Exporting alarms and diagnostic logs

**Built for:**  
System integrators, service engineers, and support personnel responsible for alarm triage and response

**Developed by:**  
Distech Controls Advanced Support Team (community project)

---

## ✨ Key Features

### 🚨 **Multi-Controller Alarm Monitoring**

- Monitor alarms from **multiple ECLYPSE Facilities IPs simultaneously**
- Centralized alarm list with per-source tagging
- Automatic merge and refresh per device
- Supports large datasets (thousands of alarms)

---

### 🔄 **Live Polling & Automation**

- Configurable auto-polling interval (seconds)
- Visual live/online indicator
- Countdown to next poll
- Manual fetch on demand
- Optimized parallel polling via RunspacePool

---

### 🔍 **Advanced Filtering & Search**

- Filter by:
  - Alarm state (Normal / Offnormal / Fault)
  - Acknowledgment state (Acked / Unacked)
- Text search across alarm messages
- Optional time-range filtering (last _N_ days)
- Show/hide inactive or hidden alarms

---

### 📄 **Paging & Performance Controls**

- Page size options: 500 → 10,000 rows
- First / Previous / Next / Last page navigation
- Designed for high-volume alarm systems
- Efficient merge logic to prevent UI freezing

---

### ✅ **Alarm Acknowledgment**

- Acknowledge:
  - Single alarm (context menu)
  - All filtered alarms (bulk)
- Confirmation feedback and status updates
- Uses standard ECLYPSE REST acknowledgment endpoints

⚠️ *Acknowledgment affects live alarm state—use responsibly.*

---

### 🧾 **Detailed Alarm Inspection**

- Expandable row detail panel with:
  - Alarm ID and source
  - Raw and local timestamps
  - State transitions
  - Event state and priority
  - Exceeding values and limits
  - Acknowledgment user/time
  - Raw JSON payload

---

### 📤 **Import & Export Tools**

- **Import alarms** from:
  - Clipboard JSON
  - JSON files
  - Drag-and-drop files
- **Export alarms to CSV** (filtered view)
- **Export application logs** to TXT
- Ideal for:
  - Post-incident analysis
  - Reporting
  - Offline review

---

### 🛠️ **Credential & Settings Management**

- Optional credential storage (Base64-encoded)
- Persistent settings in `ProgramData`
- Saved state:
  - IP list
  - Poll interval
  - Filters and options
  - Window layout
- Reset-to-defaults tool

---

### 🖥️ **Operational UI & Diagnostics**

- Collapsible panels for:
  - API configuration
  - Tools & filters
  - Row details
- Color-coded alarm and ack states
- Live status indicator
- In-memory and console logging with levels:
  - INFO, OK, WARN, ERROR, DEBUG

## 🎮 Typical Workflow

1. Add one or more ECLYPSE controller IPs
2. Enter credentials (optional save)
3. Fetch alarms manually or enable auto-polling
4. Filter and search alarms
5. Inspect alarm details
6. Acknowledge alarms as required
7. Export alarms or logs for reporting

## 🐛 Stability & Risk Considerations

- Can acknowledge alarms in bulk
- Trusts self-signed certificates (common in BAS)
- High-frequency polling may impact controller load

Always validate behavior in a test or maintenance environment first.
